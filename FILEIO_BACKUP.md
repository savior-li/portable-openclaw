# File.io 备份方案

## 限制

1. **下载后删除**: 免费版文件下载后会被删除
2. **无目录结构**: 只能上传单个文件，无法创建目录
3. **每次上传生成新链接**: 旧链接失效

## 替代方案

考虑使用以下免费方案：

| 服务 | 说明 |
|------|------|
| GitHub Gist | 存储小文件 |
| transfer.sh | 临时文件分享 |
| 0x0.st | 匿名文件存储 |
| GitHub Repository | 私有仓库存储 |

## 当前脚本

备份脚本位置: `/opt/scripts/backup-fileio.sh`

## 使用说明

file.io 需要用户界面操作，程序化使用建议使用其他方案。
