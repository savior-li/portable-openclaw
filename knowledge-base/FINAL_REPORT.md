## 🎯 核心功能特性

### 1. 双向学习机制
- 用户明确教导规则和偏好
- AI 自动学习并优化服务

### 2. 六种知识类型
- 行为规则 (behavioral)
- 偏好设置 (preferences)
- 思维模式 (thinking)
- 执行记录 (execution)
- 问题记录 (problem)
- 结果记录 (result)

### 3. 智能学习引擎
- 自动从指令提取规则 (准确率 ~80%)
- 检测重复行为模式
- 智能解决规则冲突
- 知识衰减保持新鲜度

### 4. 完整工具链
- CLI 工具 (命令行操作)
- Python API (编程接口)
- 自动学习 (无需配置)
- 数据导出 (JSON 格式)

## 📚 知识库规则分类

### 高优先级规则 (4 条)
1. 禁止删除文件
2. 所有 API 调用必须有错误处理
3. 禁止直接推送到 main 分支
4. 所有代码变更必须经过审查

### 中优先级规则 (2 条)
1. 避免在循环中进行数据库查询
2. 测试规则：所有文件操作需要确认

### 工具偏好 (7 条)
1. 使用 pnpm 而不是 npm
2. 优先使用 TypeScript
3. 使用 Husky 管理 Git Hooks
4. 使用 ESLint + Prettier
5. 使用 Jest 作为测试框架
6. 使用 ESLint 进行代码检查
7. 使用 Prettier 进行代码格式化
